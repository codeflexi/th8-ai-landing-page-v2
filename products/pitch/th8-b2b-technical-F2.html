05. Policy Engine Internals

Decoupled Logic

Business logic is separated from application code. Stored as YAML/JSON and versioned in Git.

Performance

Rules are compiled into an Abstract Syntax Tree (AST) for O(1) execution speed.

rules/procurement.yaml
# High Variance Check
- id: "RULE_PRICE_VARIANCE"
  priority: 100
  condition:
    and:
      - field: "item.price"
        operator: "gt"
        reference: "contract.price * 1.05"
      - field: "vendor.status"
        operator: "eq"
        value: "ACTIVE"
  action:
    type: "BLOCK"
    reason: "Exceeds contract by > 5%"

12. Data Schema & Entity Model

schema/core.sql
CREATE TABLE transactions (
  id UUID PRIMARY KEY,
  erp_ref_id VARCHAR(64) INDEX,
  vendor_id UUID REFERENCES vendors(id),
  amount DECIMAL(18,2),
  status VARCHAR(20),
  risk_score FLOAT,
  created_at TIMESTAMP WITH TIME ZONE
);

CREATE TABLE audit_logs (
  id UUID PRIMARY KEY,
  tx_id UUID REFERENCES transactions(id),
  actor VARCHAR(100),
  action VARCHAR(50),
  payload JSONB, -- Stores full context
  hash VARCHAR(64) -- Tamper-proof
);

Relational Core

Normalized schema for high-integrity financial records (ACID compliant).

JSONB Flexibility

Store dynamic rule results and evidence traces without schema migration.

Immutable Audit

Cryptographic hashing of logs ensures non-repudiation for compliance.

15. Tamper-Proof Audit Logging

Every decision and action is recorded in an immutable ledger. Designed for external auditors and forensic analysis.

Write-Once, Read-Many (WORM)

Logs are pushed to append-only storage (e.g., S3 Object Lock) to prevent modification.

Chain of Custody

Each log entry contains a cryptographic hash of the previous entry, forming a blockchain-like structure.

// Log Entry Structure
{
  "id": "log_001",
