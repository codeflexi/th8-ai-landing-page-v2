04. Integration Patterns

A. Asynchronous API (Preferred)

Non-blocking architecture. ERP pushes data to webhook/queue. TH8 processes and callbacks.

ERP -------- POST /po (Payload) --------> TH8 Queue
| (200 OK - Ack)
[ Processing ]
ERP <------- POST /callback (Result) -------- TH8 Worker

B. Change Data Capture (CDC)

Connect directly to DB Transaction Logs (Debezium/Kafka). Zero impact on ERP API limits.

DB Log (WAL)
1. Insert PO #1021 ...
2. Debezium captures event
3. Stream to Kafka Topic
4. TH8 Consumer reads topic

05. Policy Engine Internals

Decoupled Logic: Rules are stored as YAML/JSON, versioned in Git, and compiled to efficient AST at runtime.

rules/procurement.yaml 
# Rule: Flag High Variance
- id: PRICE_VARIANCE_CHECK
  priority: 10
  condition:
    and:
      - field: item.price
        operator: gt
        reference: contract.price * 1.05
      - field: vendor.status
        operator: eq
        value: "ACTIVE"
  action:
    type: BLOCK
    reason: "Price exceeds contract > 5%"
