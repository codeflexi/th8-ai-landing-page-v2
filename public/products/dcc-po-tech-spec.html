Policy Configuration | กำหนด Logic ของ AI (YAML)

Business Rules Map

R-01 Price Deviation Threshold: +1%, +5%
R-02 CFO Threshold Value > 500,000 THB
R-03 Vendor Match Strict String Match
R-04 Scope Check SKU in Contract List
R-05 Validity PO Date within Contract Period

"Config นี้จะถูกโหลดเข้า Policy Engine เพื่อใช้ตรวจจับความผิดปกติ"

policy_v1.0.yaml 
rules_config:
  # R-01: Price Tolerance Logic
  - id: "R-01"
    type: "price_deviation"
    thresholds:
      warning: 1.0  # >1% = Warning
      critical: 5.0  # >5% = Violation (Medium)

  # R-02: Authority Limit
  - id: "R-02"
    type: "total_amount_threshold"
    max_value: 500000
    currency: "THB"
    action: "escalate_to_cfo"

  # R-03: Vendor Verification
  - id: "R-03"
    type: "vendor_entity_match"
    strict_mode: true
    severity: "High"

  # R-04: Item Scope
  - id: "R-04"
    type: "sku_lookup"
    source: "vector_db_line_items"

  # R-05: Contract Validity Period
  - id: "R-05"
    type: "date_range_check"
    reference: "contract_effective_dates"

ERP Data Ingestion | ข้อมูล PO ที่วิ่งเข้ามา (JSON)

Input Requirements

ระบบต้องการข้อมูล PO (Purchase Order) จาก ERP (เช่น SAP, Oracle, MS Dynamics) เพื่อนำมาตรวจสอบเทียบกับสัญญา

Key Fields Required:

  • PO Header: Number, Date, Vendor ID, Currency
  • Line Items: SKU Code, Description, Quantity
  • Commercials: Unit Price, Total Amount

RAG Document Maintenance | การจัดการความรู้ใน Vector DB

Ingestion Process (Loading)

1

PDF Parsing (OCR)

Extract text, tables, and headers from contract.pdf.

2

Chunking & Metadata

Split text into smaller pieces (500-1000 tokens) and tag with metadata (Vendor, Date).

3

Vector Embedding

Convert text to vector array [0.02, -0.15, ...] via OpenAI/Cohere.
