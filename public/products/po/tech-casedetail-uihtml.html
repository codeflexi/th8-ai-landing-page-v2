What to say (ผู้บริหารเข้าใจทันที)

  • Policy เป็น “เอกสารควบคุม” ไม่ฝังในโค้ด → เปลี่ยนเวอร์ชันได้โดยไม่ deploy
  • Decision ทุกครั้ง bind กับ policy/version แบบ immutable → audit อธิบายย้อนหลังได้
  • คนตัดสินยังเป็นคน แต่ระบบทำให้ “ตัดสินบนหลักฐาน” และรับผิดชอบได้

YAML preview (read-only)

policy_id: PROCUREMENT-001
version: v3
thresholds:
  amount:
    medium: 200000
sla:
  approval_due_hours: 24
authority:
  rules:
    - condition: amount > 500000
      required_role: CFO
    - condition: amount > 200000
      required_role: COO
rules:
  - id: HIGH_AMOUNT_ESCALATION
    when: [{ field: amount, operator: ">", value: 200000 }]
    then: { decision: ESCALATE }
  - id: SLA_RISK
    when: [{ field: hours_to_sla, operator: "<", value: 24 }]
    then: { decision: ESCALATE }

Why separate statuses?

External status อยู่ในระบบหลัก (เช่น SAP/Workflow) แต่ Decision status อยู่ใน TH8 เพื่อให้ คำแนะนำ, หลักฐาน, และ การกระทำของคน ถูก audit ได้โดยไม่ไปแก้ state ซ้ำซ้อนในระบบต้นทาง

Step 3

Run Decision

กดเพื่อ trigger → ได้ recommended_decision, rule_hits, required_role และ bind หลักฐาน

Policy: PROCUREMENT-001 / v3

Recommended decision

ESCALATE

decision_required = true

Engine: deterministic

Required role

COO

Decision ID

DEC-d5eb312b

Rule hits

HIGH_AMOUNT_ESCALATION SLA_RISK

Explainability (concept)

IF amount_total > 200,000
AND hours_to_sla < 24
THEN ESCALATE
required_role = authority(amount_total)

API mapping

POST /api/decisions/trigger
payload: { case_id, policy_id, version }

Safety behavior

ถ้า case ไม่มี field ที่ policy อ้าง → rule นั้นไม่ match (ปลอดภัย) และผลลัพธ์ยัง deterministic

Talk track (CFO/COO/IT)

“สิ่งที่เราได้ไม่ใช่คำตอบจาก AI แต่เป็น recommendation ที่ตรวจสอบได้: rule hits + ผู้มีอำนาจตัดสิน + หลักฐานอ้างอิง + บันทึกเหตุผลของคน”

Why executives believe this

  • หลักฐาน “จับต้องได้” ไม่ใช่คำตอบลอย ๆ
  • Snapshot เข้า decision + audit → ตรวจย้อนหลังได้ แม้ policy/เอกสารถูกอัปเดตเวอร์ชันใหม่
  • เหมาะกับองค์กรที่ต้อง audit/ISO/SOX/ควบคุมภายใน

API mapping

GET /api/audit/case/{case_id}
Evidence lives inside DECISION_RECOMMENDED payload

Design rule

Evidence ต้อง “คลิกเปิดเอกสารได้” และต้อง “อ่านแล้วเข้าใจเหตุผลที่ match” ใน 10 วินาที
